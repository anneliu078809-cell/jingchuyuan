Project guide

靜初苑網站專案說明

這份文件是給第一次接觸程式的人看的版本。它不會逐行解釋程式碼，而是說明這個網站是什麼、每個資料夾在做什麼、聯絡表單怎麼把資料寄出去，以及日後維護時要知道哪些關鍵字。

網站類型 Astro 靜態網站
部署平台 Cloudflare Pages
表單寄信 Cloudflare Function + Resend
正式網域 jingchuyuan.com

01 / Big picture

這個專案在做什麼

這是一個「靜初苑」服務介紹網站。訪客可以閱讀首頁、服務方案、緣起、常見問題，也可以到聯絡頁加入 LINE、前往 Instagram，或填寫聯絡表單。

前台頁面

訪客看到的內容，例如首頁、服務介紹、FAQ、聯絡頁。這些頁面主要放在 src/pages。

共用版型

網站每一頁都共用上方導覽、SEO 設定、頁尾。這些集中在 src/layouts/BaseLayout.astro。

聯絡表單後端

表單不是存在資料庫，而是送到 /api/contact，再由 Resend 寄到指定信箱。

03 / Structure

資料夾和檔案怎麼看

  • src/pages/index.astro 首頁。介紹靜初苑、適合對象、預約流程。
  • src/pages/service.astro 服務方案頁。列出問事服務、牌卡占卜、手工藥香、調頻蠟燭、家族排列、身體經絡放鬆引導。
  • src/pages/about.astro 緣起頁。放靜初苑的緣起與理念。
  • src/pages/faq.astro 常見問題頁。回答第一次詢問、服務方式、費用、預約等問題。
  • src/pages/contact.astro 線上預約頁。包含 LINE、Instagram、Email 與聯絡表單。
  • src/layouts/BaseLayout.astro 所有頁面共用的外框：導覽列、頁尾、SEO、社群分享資訊、結構化資料。
  • src/components/PictureSlot.astro 圖片顯示元件。各頁面的主視覺或情境圖會用到它。
  • functions/api/contact.js 聯絡表單後端。檢查資料、驗證防機器人、限制頻率，然後呼叫 Resend 寄信。

05 / Contact form

聯絡表單怎麼運作

1

訪客填表單

表單欄位包含姓名或暱稱、想詢問的服務、偏好的聯絡方式、聯絡資料、想聊的方向。

2

前端先做基本檢查

例如必填欄位不能空白、文字不能超過限制。如果有設定 Turnstile，頁面會載入 Cloudflare 的防機器人元件。

3

送到 Cloudflare Pages Function

表單資料送到 /api/contact，由 functions/api/contact.js 處理。

4

後端做安全檢查

後端會檢查來源網址、欄位長度、必填資料、蜜罐欄位、Turnstile token，並做簡單頻率限制。

5

用 Resend 寄信

檢查通過後，後端呼叫 Resend API，把表單內容寄到 jingchuyuan1413@gmail.com。

這個專案目前沒有資料庫，也沒有後台管理頁。表單資料的保存位置就是 Email 信箱本身。如果需要查詢歷史表單，會去信箱裡搜尋。

06 / Deployment

上線和部署怎麼理解

這個網站部署在 Cloudflare Pages。平常修改程式後，會把程式推到 GitHub，Cloudflare Pages 會依設定建置網站並發布。

建置指令

npm run build

輸出資料夾

dist

本機開發網址

http://127.0.0.1:4321/

Cloudflare Pages 後台需要設定環境變數，例如 NODE_VERSION、RESEND_API_KEY、CONTACT_TO_EMAIL、CONTACT_FROM_EMAIL、PUBLIC_TURNSTILE_SITE_KEY、TURNSTILE_SECRET_KEY、CONTACT_ALLOWED_ORIGINS。

07 / SEO

Google 搜尋相關設定

專案裡有基本 SEO 設定。每一頁透過共用版型放入標題、描述、canonical URL、Open Graph 圖片、Twitter card 和 JSON-LD 結構化資料。

  • public/robots.txt：告訴搜尋引擎可以爬網站，並指定 sitemap 位置。
  • public/sitemap.xml：列出主要頁面，方便 Google Search Console 提交。
  • BaseLayout.astro：集中管理大部分 SEO 標籤。

08 / Maintenance

日後改內容時看哪裡

改服務項目或價格 看 src/pages/service.astro，服務資料集中在檔案上方的 services 陣列。
改 LINE / Instagram / Email 看 src/pages/contact.astro。如果 Email 要同步影響表單收件，也要檢查 Cloudflare 的 CONTACT_TO_EMAIL。
改導覽列或頁尾 看 src/layouts/BaseLayout.astro 的 navItems 和 footerItems。
改圖片 看 public/images。頁面裡會用 /images/... 這種路徑引用圖片。
改表單規則 前端看 src/pages/contact.astro，後端看 functions/api/contact.js。兩邊的欄位名稱和長度限制最好保持一致。
